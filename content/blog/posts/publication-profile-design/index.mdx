This site is organized as a compact research index. The main constraint is that
different kinds of work should not compete for the same layout.

## Structure

The profile separates identity, writing, papers, notes, and archived project
logs. Each section can grow independently while still feeding the same homepage
and archive views.

## Design Rule

The blog index should be stable even when article sources vary. A full HTML
article, an MDX essay, and an external link can share the same metadata:
title, summary, date, read time, tags, image, and destination.

## Why This Matters

The more the site grows, the more important it becomes to avoid manual layout
decisions for every post. A small metadata contract keeps the system predictable
while leaving enough freedom for rich articles.
